## 1분 요약 — 무엇이 달라졌나 (4가지)

1. **목표 기능 신설**: 원본엔 없던 *"그 자리에 심을 적합 수종 추천"* 을 핵심으로 구현.
   (원본은 *이미 자라는 숲을 진단*만 했고, 그 진단 기능도 **유지**했습니다 → 진단+추천 둘 다)
2. **ML을 올바르게 교체**: 원본의 "규칙을 다시 외우는 가짜 학습(순환 ML)"만 제거하고,
   임상도 실분포로 학습하는 **올바른 ML(SDM)** 을 추천에 **기본 사용**(신뢰도 비례 가중).
3. **좌표계·래스터 버그와 실행 불가 버그 다수 수정** → **데이터 없이도 즉시 실행**.
4. **사용 환경 확장**: Windows 더블클릭 실행 / 모바일 웹 / 정식 앱 연결용 REST API.

→ 자세한 "왜·어떻게"는 비유와 예시로 [변경사항_리포트.md](변경사항_리포트.md)에 풀어 썼습니다.

## 폴더째 전달(압축) 방법

작업 폴더(`wooyoung`)를 그대로 압축해 전달하면 됩니다. 단, **용량/불필요 파일 제외** 권장:

```powershell
# PowerShell — 캐시·생성물 제외하고 압축
$ex = @('__pycache__','.pytest_cache','data','*.pyc')
robocopy . ..\wooyoung_deliver /E /XD __pycache__ .pytest_cache data /XF *.pyc
Compress-Archive -Path ..\wooyoung_deliver\* -DestinationPath ..\wooyoung_deliver.zip -Force
```
- `data/`는 **합성 데이터(자동 생성)** 라 빼도 됩니다(받는 쪽에서 자동 재생성). 실데이터는 별도 전달.
- `.claude/`, `docs/research/`는 참고용이라 포함해도/빼도 무방합니다.
- 받는 사람은 압축 풀고 `pip install -r requirements.txt` → `run_app.bat` 또는 `COLAB_가이드.md`.
