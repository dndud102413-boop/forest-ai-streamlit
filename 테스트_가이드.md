# 기능별 테스트 가이드 — 무엇을 어떻게 확인하나

> 각 기능을 ① **자동 테스트(pytest)** 와 ② **직접 한 줄 실행**으로 확인하는 법을 모았습니다.
> 데이터 없이도 전부 동작합니다(합성 데이터 자동 생성).

## 0. 한 번에 전체 점검
```cmd
pip install -r requirements.txt
python -m pytest                 REM 전체 53개 (느린 SDM 2개 포함, ~25초)
python -m pytest -m "not slow"   REM 빠른 51개만 (~5초)
python -m pytest -v              REM 어떤 테스트가 무엇을 검증하는지 이름까지
```
모두 `passed`면 핵심 로직이 정상입니다. 개별 기능은 아래에서 확인하세요.

## 1. 사진 GPS 추출 (EXIF) — `exif_gps.py`
- **자동**: `python -m pytest tests/test_exif.py -v`
  (남/서반구 부호, 누락 처리, 바이트 입력, 좌표 라운드트립 검증)
- **직접**:
  ```python
  from PIL import Image
  from forest_reco.exif_gps import write_gps_exif, extract_gps
  Image.new('RGB',(32,24),(0,128,0)).save('p.jpg','jpeg')
  write_gps_exif('p.jpg', 37.8813, 127.7298, 'geo.jpg')   # 테스트용 GPS 심기
  print(extract_gps('geo.jpg'))      # → lat=37.8813, lon=127.7298, ok=True
  print(extract_gps('p.jpg').reason) # → 'EXIF 메타데이터 없음'(정상)
  ```
- **실제 휴대폰 사진**으로: 갤럭시에서 위치 켜고 찍은 사진 경로를 `extract_gps('내사진.jpg')` 에 넣어 위·경도가 나오는지.

## 2. 좌표계·래스터 샘플링 — `geo.py`
- **자동**: `python -m pytest tests/test_geo.py -v`
  (WGS84↔5179 변환, 경계 밖→예외, nodata→예외, 경사 45° 검증, 평지=NaN)
- **직접**:
  ```python
  from forest_reco.geo import transform_point, aspect_to_direction
  print(transform_point(127.73, 37.88, 'EPSG:4326', 'EPSG:5179'))  # 백만 단위 좌표
  print(aspect_to_direction(180), aspect_to_direction(float('nan')))  # 남향, 평지
  ```

## 3. 현재 숲 진단 — `diagnosis.py`
- **자동**: `python -m pytest tests/test_diagnosis.py -v`
- **직접**:
  ```python
  from forest_reco.diagnosis import diagnose_stand
  d = diagnose_stand({'수종':'소나무','경급':'대경목','영급':'6영급','밀도':'밀'},
                     {'고도':400,'경사':35,'향':'남향'})
  print(d.as_dict())   # 노령·과밀 소나무 → 병해충 취약 '높음', 관리 '높음' + 근거
  ```

## 4. 수종 추천(핵심) — `recommender.py`
- **자동**: `python -m pytest tests/test_recommender.py -v`
  (고지대→한랭수종, 저지대→난대수종, 목적별 랭킹 변화, 경험가점, 고도 경계 역전 없음)
- **직접**:
  ```cmd
  python scripts/run_demo.py --lat 37.95 --lon 127.66 --goal 탄소흡수 --mock
  python scripts/run_demo.py --lat 37.66 --lon 127.55 --goal 경관조경 --mock
  ```
  고도가 높을수록 잣나무/신갈나무/분비나무 등 한랭 수종이, 목적을 바꾸면 상위 구성이 달라지는지 확인.

## 5. 기후대 판정 — `climate.py`
- **자동**: `python -m pytest tests/test_recommender.py -k climate -v`
- **직접**:
  ```python
  from forest_reco.climate import classify_zone
  print(classify_zone(34.3, 50).zone)    # 난대
  print(classify_zone(37.7, 1600).zone)  # 한대(아고산)
  ```

## 6. ML 모델(SDM) — `sdm.py`
- **자동**: `python -m pytest tests/test_sdm.py -v`
  (학습/예측, 확률 합=1, 신뢰도 비례 가중, **앙상블(hgb+rf)**, 빈 DB 존중)
- **직접**(신뢰도·가중치 확인):
  ```python
  from forest_reco.pipeline import analyze, DataSources
  r = analyze(lat=37.95, lon=127.66, sources=DataSources(use_mock=True),
              use_sdm=True, explain=False)
  print('ML 사용:', r['sdm_used'], '신뢰도f1:', r['sdm_quality'], '가중치:', r['sdm_weight_applied'])
  ```
  (합성데이터는 f1이 낮아 가중치가 자동 축소됩니다 — 정상. 실데이터에서 올라감)
- **앙상블 켜보기**: 위 호출 전에 `os.environ['FOREST_RECO_SDM_ALGOS']='hgb,rf'`

## 9. 모바일 앱 — `app/streamlit_app.py`
- **직접 실행**: `run_app.bat`(또는 `streamlit run app/streamlit_app.py`) → 브라우저에서
  좌표 직접 탭 → "분석" → 입지/진단/추천/설명 4섹션이 뜨는지. 휴대폰은 같은 와이파이로
  `http://<PC-IP>:8501`.
- **자동(헤드리스 렌더 점검)**:
  ```python
  from streamlit.testing.v1 import AppTest
  at = AppTest.from_file('app/streamlit_app.py', default_timeout=120); at.run()
  [b.click() for b in at.button if '분석' in b.label]; at.run()
  print('에러:', at.exception, '| 섹션:', [s.value for s in at.subheader])
  ```

## 10. REST API — `app/api.py`
- **직접 실행**: `run_api.bat` → 브라우저 `http://localhost:8000/docs` (Swagger에서 직접 호출)
- **명령행 점검**:
  ```cmd
  curl http://localhost:8000/health
  curl -X POST http://localhost:8000/analyze -H "Content-Type: application/json" ^
       -d "{\"lat\":37.95,\"lon\":127.66,\"goal\":\"탄소흡수\"}"
  ```
  사진 업로드: `/docs`의 `POST /analyze/photo`에서 위치정보 포함 사진 업로드 → 추천 JSON.

## 11. 실데이터 점검(적용 전 필수) — `scripts/prepare_data.py`
